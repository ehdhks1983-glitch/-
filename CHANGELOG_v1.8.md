# GIF Maker Pro — v1.8.0 (고화질 + 로딩 최적화)

화질을 올리면서 **용량(=로딩)도 같이 잡는** 변경. 대부분은 용량이 늘지 않거나 오히려 줄면서 화질이 좋아집니다.

## 🎚️ 화질 프리셋 (영상 → 움짤) — 한 번만 선택
포맷 아래 세그먼트 버튼: **🟢 최고화질 / 🔵 균형(기본) / 🟡 빠른로딩**
- **🟢 최고화질**: 오차확산 디더(`sierra2_4a`) + `stats_mode=full` → 그라데이션 매끈, 무압축
- **🔵 균형**: `bayer:bayer_scale=3` + `diff_mode=rectangle` + gifsicle `--lossy=30`
- **🟡 빠른로딩**: 위 + **fps 15 상한** + `--lossy=60` → 최소 용량

## 🆓 공짜 화질 이득 (모든 GIF에 기본 적용)
- **`diff_mode=rectangle`**: 정지 배경 떨림(crawl) 제거 + 압축↑
- 디더 `bayer_scale` 5 → 3 (미세, 밴딩↓)
- **이미지 → GIF 글로벌 팔레트**: 프레임마다 따로 양자화하던 걸 공용 팔레트 1개로 → **프레임 간 색 깜빡임 제거**

## ⚙️ 로딩(용량) 잡기
- `optimizer.polish_gif()`: 균형/빠른로딩에서 gifsicle `--lossy -O3`를 **항상** 적용(있을 때, 더 작아질 때만 교체)
- 빠른로딩 fps 상한

## 💡 WebP 추천 안내
- 영상탭에 "블로그·웹은 WebP 추천 — 같은 화질에 용량 30~50%↓(로딩 빠름)" 힌트

## ✅ 검증 (실측, gifsicle 미설치 환경)
- 같은 2초 소스: **최고화질 274KB / 균형 243KB / 빠른로딩 189KB** (의도대로 best>균형>빠른로딩)
- 이미지→GIF 공용 팔레트 저장 OK(불투명/투명), 회귀(engine·mp4·watermark·grid·gui) 전부 통과
- ※ 사용자 PC는 gifsicle 번들 → 균형/빠른로딩 용량이 여기보다 더 줄어듦
