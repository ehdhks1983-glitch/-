## 1. 이 프로젝트가 왜 생겼나 (배경)

### 최초 아이디어
1688(중국 도매) 상품을 가져와서 AI로 한국형 상세페이지를 만들어주는 SaaS로 출발.

### 피봇 히스토리 (중요 — 같은 실수 반복 방지)
1. **1688 스크래핑 시도** → 로그인 지옥, 봇 차단, 법적 회색지대로 **포기**
2. **국내 도매 사이트로 전환** → 도매꾹/도매매는 **공식 Open API 존재** (합법·안정)
3. **"상세페이지 제작"에서 "엔드투엔드 자동등록"으로 확장**
   - 단순 상세페이지 제작툴은 경쟁자 많음 (카페24 에디봇 등)
   - "도매꾹 소싱 → AI 가공 → 쿠팡 자동등록"은 **경쟁자 거의 없는 원툴**
4. **현재**: 도매꾹 → 쿠팡 자동등록 엔진(Slice 1) 완성, 실전 검증 대기

### 제품 정체성
> "도매꾹 상품번호만 넣으면 쿠팡에 자동 등록까지 5분에 끝"

타겟: 월매출 1천만 이상의 검증된 셀러 (이미 샵플링/스피드고전송기 같은 유료툴 쓰는 사람).

## 3. 핵심 기술 결정 사항 (왜 이렇게 했는지)

### 왜 API 방식인가 (엑셀 방식이 아니라)
도완님이 써본 경쟁 업체는 "도매매에서 엑셀 다운 → 프로그램에 업로드 → 등록" 방식이었음.
우리는 **도매꾹 API 직접 호출** 방식. 차이:
- API: 상품번호만 알면 실시간 조회·등록 (엑셀 다운 불필요)
- 엑셀: 사이트에서 대량 선택 가능하지만 다운로드 단계 필요

**⚠️ 미해결 논의**: 대량등록을 어떻게 할지 아직 안 정함. 3가지 옵션:
- A. 상품번호 여러 개 직접 입력
- B. 도매꾹 검색 API로 자동 수집
- C. 엑셀 업로드 (경쟁사 방식)
→ 도완님이 "대량등록 되냐"고 물어본 상태. **이게 다음 핵심 결정.**

### 가격 계산 — 3가지 모드
도완님 요구: "퍼센트나 최소마진 몇% 식으로". 그래서:
- multiply: 도매가 × N배
- add_margin: 도매가 + N%
- min_margin: 최소 마진율 N% 보장
GUI에서 드롭다운으로 선택 + 즉석 수치 변경 가능.

### 이미지 — 크롤링(URL 직접 전달) 방식
도완님 결정. 도매꾹 이미지 URL을 쿠팡에 그대로 전달.
**리스크**: 쿠팡이 외부 도메인 이미지를 거부할 수 있음 → 그럴 경우 자체 호스팅(S3 등) 필요. (Slice 2 고려사항)

### 카테고리 — 쿠팡 자동 추천 API
도완님 결정 ("자동으로 되던데 다른 프로그램은"). 상품명으로 쿠팡이 카테고리 추천.

### 정보 최대한 수집 (스냅샷)
도완님 요구: "정보 최대한 모으자". 그래서 모든 처리 단계를 JSON으로 저장(`snapshots/`).
디버깅·재시작·통계에 활용.

## 5. 절대 다시 시도하면 안 되는 것 (삽질 방지)

- ❌ **1688 스크래핑** — 로그인 지옥. 이미 포기함. 다시 가지 말 것.
- ❌ **쿠팡 결제/라이선스 로직을 봇에 직접 삽입** — EXE 디컴파일로 크랙 위험. 중계 서버 필수.
- ❌ **Playwright로 도매꾹/쿠팡 크롤링** — API 있는데 굳이. 봇 감지 위험.
