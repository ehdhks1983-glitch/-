# SellerFit 개발 로드맵 (기획안)

> 완료된 작업 + 앞으로 개발할 작업 전체 청사진.

---

## 제품 비전

**도매꾹 상품번호 입력 → AI 가공 → 쿠팡 자동등록까지 원클릭.**

경쟁 우위: 소싱+가공+등록을 한 번에 하는 **엔드투엔드 원툴** (경쟁사는 등록 OR 카피 하나만).

### 가격 모델 (예정)
| 플랜 | 월 가격 | 기능 |
|---|---|---|
| 스타터 | 49,000원 | 단일 등록, 월 50건 |
| 프로 | 99,000원 | 대량 등록, 월 200건 |
| 비즈 | 199,000원 | 무제한 + 다채널(스토어 추가) |

---

## 개발 단계 (Slice 방식)

작은 단위로 "작동하는 제품"을 단계적으로 완성. 각 Slice 끝에 실제 동작 검증.

---

### ✅ Slice 1 — 단일상품 등록 (코드 완성 / 검증 대기)

**목표**: 도매꾹 상품 1개 → 쿠팡 등록

**완료된 기능:**
- 도매꾹 API 상품 조회 + XML 파싱
- 쿠팡 WING API 클라이언트 (HMAC 인증)
- 가격 계산 3모드 (배수/마진율/최소마진)
- 카테고리 자동 추천 + 필수옵션 자동 채움
- 반품지/출고지 자동 조회 + 캐시
- 이미지 접근성 검증
- 쿠팡 등록 payload 조립 + 로컬 검증
- 단계별 스냅샷 저장
- CLI + GUI (조회/등록 2단계, 가격조정, 실시간 로그)

**남은 일:**
- [ ] 도완님 실제 키로 1개 등록 성공 검증
- [ ] 쿠팡 실제 응답 기반 payload 보정
- [ ] 도매꾹 실제 XML 형식 확인 (이미지 파싱 등)

---

### 🔜 Slice 2 — 대량 등록 (다음 작업)

**목표**: 여러 상품을 한 번에 등록

**핵심 결정 필요 (도완님과 합의):**
상품번호 리스트를 어떻게 확보할지 —
- 옵션 A: 상품번호 직접 여러 개 입력 (텍스트 붙여넣기) — 가장 간단
- 옵션 B: 도매꾹 검색 API로 자동 수집 — 편리, API 확인 필요
- 옵션 C: 엑셀 업로드 (경쟁사 방식) — 도매꾹 엑셀 양식 확인 필요

**개발 항목:**
- [ ] `pipeline_service.py`에 `fetch_and_register_batch()` 추가
- [ ] 도매꾹 API throttle (분당 180회 제한 대응 — 호출 간 간격 조절)
- [ ] 중간 실패 처리 (실패한 것만 재시도, 건너뛰기 옵션)
- [ ] GUI: 진행률 바 + 성공/실패 카운트 + 실패 목록
- [ ] 배치 결과 리포트 (CSV로 내보내기: 상품번호 / 성공여부 / 쿠팡ID / 에러)

**기술 고려:**
- 1개당 3~7초 → 100개면 5~12분 (백그라운드 스레드)
- 스냅샷이 상품마다 쌓이므로 디스크 관리 필요할 수 있음

---

### 🔜 Slice 3 — AI 카피 + 상세페이지 템플릿

**목표**: 단순 이미지 나열이 아닌, AI가 만든 매력적인 상세페이지

**개발 항목:**
- [ ] Claude API 연동 모듈 (`ai_copywriter.py`)
- [ ] 상품 정보 → 상세페이지 카피 생성 (제목/소개/특징/마무리)
- [ ] A/B 여러 버전 생성 후 선택
- [ ] 카테고리별 HTML 템플릿 (처음엔 3~5종, 점차 30종)
- [ ] 템플릿 + 카피 + 이미지 조립 → 쿠팡 contents에 삽입
- [ ] GUI: 생성된 상세페이지 미리보기

**콘텐츠 규칙 (도완님 원칙):**
- 의사/병원/전문의 단어·이미지 절대 금지
- 팩트 기반, 과장 금지
- 태그 정확히 10개 (블로그 연계 시)

---

### 🔜 Slice 4 — 옵션 다건 처리

**목표**: 색상/사이즈 등 여러 옵션 상품을 옵션 그대로 등록

**개발 항목:**
- [ ] 도매꾹 옵션 파싱 정교화 (현재는 단일상품으로 축소 중)
- [ ] 도매꾹 옵션 → 쿠팡 items 다건 매핑
- [ ] 옵션별 가격 차등 (도매꾹 옵션가 → 쿠팡 옵션가)
- [ ] 옵션별 재고 설정
- [ ] payload_builder의 `_build_items` 다건 대응

---

### 🔜 Slice 5 — 빌드 / 배포 / 라이선스

**목표**: 일반 사용자가 설치해서 쓰는 EXE 제품

**개발 항목:**
- [ ] PyInstaller 빌드 스크립트 (`build.py`)
  - hidden imports, --collect-all, .env 제외 확인
  - sys.frozen 분기 검증
- [ ] Inno Setup 인스톨러
- [ ] 라이선스 시스템 (Keygen + 포트원 + Vercel 중계 — 기존 도완님 인프라 활용)
  - ⚠️ 결제/라이선스 로직 봇에 직접 삽입 금지 (크랙 위험)
- [ ] 자동 업데이트 (GitHub 릴리스 기반)
- [ ] 첫 실행 설정 마법사 (API 키 입력 안내)

**UX 원칙 (도완님):**
- 비개발자도 따라할 수준 (큰 버튼, 단계 번호, 신호등 색상)
- 에러는 사람 말로 ("HTTP 401" → "비밀번호가 틀렸어요")
