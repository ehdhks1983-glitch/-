# 에이전트 작업 매뉴얼 (skill.md)

이 문서는 콘텐츠 생성 AI가 따라야 하는 **동작 규칙**이다. 코드가 아니라 운영 지침이며,
`content_engine`이 프롬프트에 이 원칙들을 주입/강제한다.

## 1. 절대 금지 (Red Line)
- 자동 팔로우/좋아요/댓글/DM 등 **자동 인게이지먼트 금지**.
- reCAPTCHA·보안문자 **우회 시도 금지** (감지 시 자동 중단 + 사용자 알림).
- 타 계정/외부 사이트 크롤링·스크래핑 금지.
- 대량 무검수 살포(스팸) 금지.
- 봇 탐지 우회(Stealth) 금지 — 공식 API 노선엔 불필요하며 위험.
- 인스타 본문/댓글 **외부 링크 도배 금지**(제휴링크 자동삽입 미포함).

## 2. 콘텐츠 규칙 (강제)
- **금지어:** `의사`, `병원`, `전문의` 및 가운·청진기 등 의료 권위 연상 표현 금지.
  개인 경험담 + 객관적 팩트 위주로 작성. (사용자 추가 금지어도 함께 차단)
- **해시태그:** 인스타 권장 **5개**(3~7 범위). 10개 이상 스팸 신호 → 자동으로 개수 강제.
  `#팔로우`, `#좋아요반사` 등 스팸 태그 및 금지어 포함 태그 자동 제거.
- **이미지 비율:** 썸네일 1:1 · 피드 4:5(또는 1:1) · 릴스 9:16. (블로그용 3:2 미사용)
- **캡션:** 첫 125자에 핵심 키워드 프론트로딩. 최대 2200자.

## 3. 게시 절차 (반자동)
1. 텍스트 생성(기획 → 캡션/해시태그) → 규칙 후처리.
2. 이미지 확보(AI 생성 또는 사용자 업로드) → 비율 보정.
3. 공개 URL 호스팅(Cloudinary, 실패 시 ImgBB) → URL 접근성 검증(9004 예방).
4. **미리보기 → 사용자 승인** (필수 컨펌 단계, 완전 무인 금지).
5. 게시: 컨테이너 생성 → `status_code=FINISHED` 폴링(9007 예방) → `media_publish`.
6. 실패 시 재시도(3회) 후 중단 + 사용자 알림 + 로그.

## 4. 안전 가드
- 24시간 API 100건 한도 인지 + 자체 보수 상한(`MAX_POSTS_PER_DAY`).
- 예약 발행은 ±15분 지터로 로봇 패턴 회피. PC/프로그램이 켜져 있을 때만 작동.
- 토큰(~60일) 만료 추적 및 자동 갱신, 인증/보안 오류 시 자동 중단 + 재인증 안내.
